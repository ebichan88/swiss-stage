# アニメーションシステム

## 1. 基本方針

**「動きは控えめに、フィードバックは確実に」**

- 大会運営ツールであり、演出よりも操作の確実性・速度を優先する
- アニメーションは「状態が変わったことを伝える」目的のみに使う
- `prefers-reduced-motion` を尊重する(MUIのデフォルトに従う)

---

## 2. 標準値

| 項目 | 値 |
|------|----|
| 短(ホバー・押下) | 150ms |
| 中(ダイアログ・Snackbar・展開) | 250ms |
| 長(ページ遷移) | 使用しない(遷移アニメーションなし) |
| イージング | MUIデフォルト(`theme.transitions.easing`)のみ使用。独自cubic-bezier禁止 |

---

## 3. 使ってよいアニメーション

| シーン | 動き |
|-------|------|
| ボタンホバー/押下 | MUIデフォルト(ripple含む)をそのまま |
| ダイアログ表示 | MUI `Dialog` デフォルト(fade + scale) |
| Snackbar | MUI デフォルト(slide) |
| アコーディオン展開(順位表の詳細等) | MUI `Collapse` |
| 結果反映のハイライト | 更新された行の背景を `primary.light` → 透明へ 1秒フェード(「更新された」ことを伝える) |
| ローディング | `CircularProgress` / `Skeleton`(一覧の初回読み込みはSkeleton) |
| 順位カードの初期表示フェード(共有ページ・管理画面の順位メニュー) | MUI `Fade`(中速250ms)を各カード/行に適用し、`transitionDelay` で1件あたり60ms程度ずつ段階表示。**初回マウント時のみ**(再取得・更新時は再フェードしない。順位の並び替えが「動いて見える」ことを避けるため) |

---

## 4. 禁止事項

- ページ全体のトランジション演出
- 無限ループするアニメーション(ローディング表示を除く)
- パララックス・スクロール連動演出
- 順位表の並び替えアニメーション(FLIP等) — 順位が「動いて見える」ことは誤読リスクになるため、更新時は即時再描画 + ハイライトのみ
- framer-motion 等アニメーションライブラリの追加(MUI内蔵で完結させる)
