# デザインシステム概要(クイックスタート)

> UI実装を始める前にまずこのドキュメントを読むこと。
> 詳細は 01〜04 の各ドキュメントを参照。

---

## 1. デザインコンセプト

**「大会会場で、誰でも、迷わず使える」**

- 利用者は20代〜70代まで幅広い(囲碁・将棋大会の参加者層)
- 会場は騒がしく、時間に追われている → **一目でわかる・押し間違えない**を最優先
- 和の競技(囲碁・将棋)にふさわしい、落ち着いた品のある見た目

### 3原則

1. **明快**: 1画面1目的。装飾より情報の判別しやすさ
2. **堅実**: 派手なアニメーションより確実なフィードバック
3. **信頼**: 対局結果・順位という「間違えてはいけない情報」を扱う緊張感のある正確な表示

---

## 2. 実装の基本(これだけ守れば大きく外れない)

- スタイルはすべて **MUIテーマ(`frontend/src/theme/index.ts`)経由**で適用する
- 色・余白・角丸をコンポーネント内でハードコードしない(`sx={{ color: '#123456' }}` 禁止 → `theme.palette.*` を使う)
- スペーシングは `theme.spacing()`(8px基準)のみ使用
- タイポグラフィは `<Typography variant="...">` のみ使用(生の `<h1>` や fontSize 直指定禁止)
- 新しいUIパターンが必要になったら、まず `02_component_design.md` に定義を追加してから実装する

## 4. 最重要画面の設計優先度

1. **共有ページ(スマホ)**: 参加者全員が使う。最優先で磨く
2. **結果入力**: 押し間違い防止が最重要(確認ステップ必須)
3. **順位表**: 大人数でも見やすい表デザイン
4. 運営者管理画面: 機能性優先でMUI標準に寄せてよい
