# デザイン原則(カラー・タイポグラフィ・スペーシング)

> これらの値は `frontend/src/theme/index.ts` に実装し、常に同期させること。

---

## 1. カラーパレット

### プライマリ / セカンダリ

| トークン | 値 | 用途 |
|---------|----|------|
| primary.main | `#1B5E43`(深緑) | 主要アクション、ヘッダー。碁盤・盤面を連想させる落ち着いた緑 |
| primary.dark | `#0F3D2A` | ホバー・アクティブ |
| primary.light | `#E8F2ED` | 選択状態の背景 |
| secondary.main | `#8D6E3F`(木目の茶) | 補助的アクセント(バッジ等)。多用しない |

### セマンティックカラー

| トークン | 値 | 用途 |
|---------|----|------|
| success.main | `#2E7D32` | 勝ち、確定済み、成功トースト |
| error.main | `#C62828` | 負け、削除、エラー |
| warning.main | `#EF6C00` | 未入力警告、制約緩和警告 |
| info.main | `#1565C0` | 情報通知 |

### ニュートラル

| トークン | 値 | 用途 |
|---------|----|------|
| text.primary | `#1A1A1A` | 本文 |
| text.secondary | `#5F6368` | 補足テキスト |
| background.default | `#FAFAF7`(わずかに温かみのある白) | ページ背景 |
| background.paper | `#FFFFFF` | カード・表 |
| divider | `#E0E0DC` | 罫線 |

### カラー使用ルール

1. 勝敗の表現は**色だけに頼らない**(○/●記号・文字を併記。色覚多様性対応)
2. テキストと背景のコントラスト比は WCAG AA(4.5:1)以上
3. 上記トークン以外の色を追加する場合は本ドキュメントに追記してから使う

---

## 2. タイポグラフィ

フォント: `"Noto Sans JP", "Hiragino Sans", "Yu Gothic", sans-serif`(数字は `font-feature-settings: "tnum"` で等幅表示 — 順位表の桁揃え用)

| variant | サイズ / 太さ | 用途 |
|---------|-------------|------|
| h1 | 2rem / 700 | ページタイトル(大会名) |
| h2 | 1.5rem / 700 | セクション見出し |
| h3 | 1.25rem / 600 | カード見出し |
| body1 | 1rem / 400 | 本文 |
| body2 | 0.875rem / 400 | 補足 |
| caption | 0.75rem / 400 | メタ情報(更新時刻等) |

- **共有ページ(スマホ)の本文は最低16px**(高齢参加者への配慮 + iOS自動ズーム防止)
- 卓番号・順位など「会場で瞬時に読む数字」は h2 相当以上の大きさで表示

---

## 3. スペーシング

- 基準: **8pxグリッド**(`theme.spacing(1)` = 8px)
- 使用可能な値: 4(0.5), 8(1), 16(2), 24(3), 32(4), 48(6), 64(8)
- コンポーネント内部の余白: 16px、カード間: 16px、セクション間: 32px
- タップターゲット: **最小44x44px**(結果入力ボタンは56px以上)

---

## 4. その他トークン

| 項目 | 値 |
|------|----|
| 角丸 | 8px(カード・ボタン共通)。`borderRadius: 2`(theme単位) |
| 影 | elevation 0〜2 のみ使用(フラット寄り)。ダイアログのみ8 |
| 罫線 | 1px solid divider |
| ダークモード | MVPでは対応しない(テーマ分岐を書かない) |
