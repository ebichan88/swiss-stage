# レイアウトシステム

## 1. ブレークポイント(MUIデフォルトを使用)

| 名前 | 幅 | 想定デバイス |
|------|----|-------------|
| xs | 0〜 | スマホ(縦) — **共有ページの基準** |
| sm | 600〜 | スマホ(横)・小型タブレット |
| md | 900〜 | タブレット・小型PC — **運営画面のサイドバー出現点** |
| lg | 1200〜 | PC — コンテナ最大幅 |

- デザインは xs(375px)から組み立てる(モバイルファースト)
- `useMediaQuery(theme.breakpoints.up('md'))` で分岐。生のpx値でのメディアクエリ禁止

---

## 2. ページ構造

### 運営者画面(AppLayout / TournamentLayout)

```
┌──────────────────────────────┐
│ AppBar(ロゴ / 大会名 / ユーザーメニュー)      │
├────────┬─────────────────────┤
│ Sidebar │  コンテンツ(max-width: 1200px)     │
│ (md以上) │  padding: 24px                    │
│         │                                   │
└────────┴─────────────────────┘
  md未満: Sidebarの代わりに下部固定タブ(BottomNavigation)
```

- サイドバー項目: 概要 / 参加者 / ラウンド / 順位 / 設定
- コンテンツは `Container maxWidth="lg"`

### 共有ページ(SharedLayout・スマホ優先)

```
┌──────────────────┐
│ ヘッダー(大会名 + 第Nラウンド) │ ← sticky
│ タブ(組み合わせ / 順位表)     │ ← sticky
├──────────────────┤
│ コンテンツ(max-width: 600px  │
│  中央寄せ、padding: 16px)     │
└──────────────────┘
```

- ナビゲーション要素は最小限(参加者は「見る・入力する」だけ)

---

## 3. グリッドと配置

- 一覧のカード配置: `Grid` で xs=12 / sm=6 / md=4
- フォーム: 1カラム縦積み(max-width: 600px)。2カラムフォーム禁止(入力順が迷子になる)
- 表(順位表・参加者一覧): 横スクロールを許容する場合は表自体を `overflow-x: auto` のコンテナで包む(ページ全体を横スクロールさせない)

---

## 4. 余白ルール

| 場所 | 値 |
|------|----|
| ページ上下 | 24px(スマホ16px) |
| セクション間 | 32px |
| カード内 | 16px |
| フォーム項目間 | 16px |
| 固定フッター(結果入力の送信ボタン等)の高さ | 72px + safe-area-inset-bottom |

- スマホの主要アクション(結果送信)は**画面下部固定**(親指が届く位置)
