# パフォーマンス最適化設計書

## 1. 目標値

| 指標 | 目標 | 備考 |
|------|------|------|
| 共有ページ初回表示 | 3秒以内(Slow 4G) | 会場の回線が悪い前提 |
| API応答(参照系) | p95 < 300ms | |
| API応答(マッチング生成) | 300名規模で < 5秒 | 進行が止まらないこと |
| 同時アクセス | 300名が順位表を同時閲覧して破綻しない | 大会当日ピーク |
| バンドルサイズ(初期JS) | gzip後 300KB以下 | MUI使用のため要監視 |

---

## 2. 負荷特性の理解(最重要)

Swiss Stage の負荷は**大会当日のラウンド確定直後にスパイクする**:

- ラウンド確定 → 参加者全員(最大300名)が一斉に組み合わせ・順位表を見る
- それ以外の時間はほぼ無負荷

したがって最適化の焦点は「**参照系APIを軽く・キャッシュ可能にする**」こと。

---

## 3. バックエンド最適化

1. **共有ページ集約のインメモリキャッシュ**(Caffeine、実装: `application/service/SharedViewCache`):
   - キー: shareToken → `SharedTournamentDto`、TTL 30秒(`app.cache.shared.ttl-seconds`)
   - 更新系操作(結果入力・ラウンド生成/確定・大会/参加者更新・トークン再発行・削除)で明示的にevict
   - 300名同時アクセスでもDynamoDBへのQueryは数回に抑える(同一キーの同時ロードはCaffeineが1回に絞る)
2. **DynamoDBアクセス**: 大会データは PK 1回の Query で全件取得(AP7)。N+1的な GetItem 連打禁止
3. **マッチング計算**: 300名でもミリ秒オーダーで終わるアルゴリズムだが、バックトラック爆発を防ぐため探索上限を設ける(上限到達時は制約緩和へ)
4. **JVM**: t3.micro(1GB)のため `-Xmx512m`。起動時にヒープ不足で落ちないことを確認
5. **レスポンス圧縮**: `server.compression.enabled=true`(共有ページ集約は300名で約90KB → gzipで約1/8。会場の低速回線対策)

---

## 4. フロントエンド最適化

1. **コード分割**: ルート単位で `React.lazy`。共有ページ(参加者向け)は運営画面のコードを読み込まない
2. **MUIのimport**: `import { Button } from '@mui/material'` 形式でOK(Viteのtree-shakingが効く)。`@mui/icons-material` はアイコン個別import必須
3. **ポーリング間隔**: 共有ページの自動更新は30秒(それ以下にしない)。タブ非アクティブ時は停止(TanStack Queryのデフォルト挙動を利用)
4. **リスト表示**: 300行程度の順位表は仮想化不要(まず素直に描画し、問題が出たら検討)
5. **静的アセット**: `Cache-Control: max-age=31536000, immutable`(Viteのハッシュ付きファイル名前提)。`index.html` は `no-cache`

---

## 5. やらないこと(過剰最適化の禁止)

- WebSocket / SSE によるリアルタイム配信(ポーリングで十分)
- CDN導入(MVP後に検討)
- SSR / プリレンダリング(SPAで問題ない。SEOは重要要件ではない)
- 計測せずに行う最適化(必ず `15_performance_monitoring.md` の計測とセットで行う)

---

## 6. 性能テスト

- リリース前に一度、負荷テストを実施: 共有ページ集約APIに300並行リクエスト(`k6` 使用)
- シナリオ: ラウンド確定直後を模擬(キャッシュコールド状態から開始)
- 合格基準: エラー率0%、p95 < 1秒
- スクリプトと手順: `perf/README.md`(データ投入 `perf/seed-shared.mjs` → `perf/k6-shared-spike.js`)
- 計測実績(2026-07-15、ローカル1台にクライアント・バックエンド・DynamoDB Local同居の保守的条件):
  300名の第1ラウンド生成 4.5秒(目標5秒以内)/ 300 VU × 5回 = 1,500リクエストでエラー0%、p95 ≈ 1.0秒
