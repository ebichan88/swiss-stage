# Swiss Stage - コンセプト・要求定義

> 囲碁・将棋大会の運営をもっとシンプルにし、参加者と運営者の双方が大会そのものを楽しめる環境を提供する。

---

# 1. プロジェクト概要

Swiss Stage は、囲碁・将棋大会向けの大会運営プラットフォームである。

単なるスイス方式の対戦組み合わせツールではなく、

* 大会準備
* 大会当日の運営
* 大会終了後のデータ管理

までを一貫して支援することを目的とする。

本プロジェクトでは、まず個人戦を対象としたMVPを開発し、その後団体戦や大会管理機能を拡張していく。

---

# 2. プロジェクト背景（Origin）

本プロジェクトは、実業団囲碁大会への参加経験から生まれた。

大会運営では、

* 紙による受付
* 対局カードの記入
* 対局カードの提出
* 手作業による結果転記
* 手作業による順位集計
* 手作業による対戦組み合わせ

など、多くの工程が人手で行われている。

以前、Google Spreadsheet・Google Forms・Google Apps Script を利用して、

* 対局結果送信
* 自動集計
* スイス方式マッチング

を実現するシステムを作成したところ、

特に

**「対局結果をスマートフォンから送信できる」**

という体験が非常に高く評価された。

その経験から、

「マッチングの自動化」だけではなく、

大会運営全体をデジタル化する価値があると考え、本プロジェクトを開始した。

# 3. ビジョン

Swiss Stage は、

**「大会といえば Swiss Stage」**

と認知されるサービスを目指す。

大会運営者にとっては、

と言われる存在になり、

大会へ参加するときに当たり前に利用するサービスになることを目標とする。

# 7. MVP

MVPでは、

**大会当日の運営負荷を減らすこと**

を最優先とする。

対象機能

* Googleログイン
* 大会作成
* 参加者管理
* CSVインポート
* スイス方式マッチング
* 対局結果入力
* 順位計算
* 順位表示
* 大会共有

---

対象外

* 団体戦
* エントリーサイト
* 決済
* ネイティブアプリ

エントリーについては、Fillout など既存SaaSとの併用を想定する。

# 8. プロダクト原則

Swiss Stage は以下を重要視する。

## 正確性

マッチング・順位計算は常に正しくなければならない。

利便性よりも正確性を優先する。

---

## シンプルさ

大会運営経験が少ない人でも利用できるUIを目指す。

---

## デジタルファースト

紙による運営をできる限り削減する。

---

## 大会運営全体を支援する

Swiss Stage はマッチングツールではない。

大会運営全体を支援するプラットフォームである。

---

## 段階的に成長する

まずはMVPを完成させる。

その後、利用者のフィードバックをもとに機能を追加していく。

---

# 9. セキュリティ・プライバシー

Swiss Stage は企業名・氏名などの個人情報を取り扱う。

そのため、以下を基本方針とする。

* 必要最小限の個人情報のみ保持する
* 利用目的を明確にする
* 利用規約・プライバシーポリシーを整備する
* 適切な認証・認可を実装する
* 大会公開範囲を設定できるようにする
* 主催者が大会データを削除できるようにする

個人情報保護は機能ではなく、プロダクト品質の一部として扱う。

## インフラ（AWS）

### 構成概要

```
[ ユーザー ] 
    ↓
[ Route53 ] → [ ALB ] → [ EC2 (t3.micro) ]
                            ↓
                        [ DynamoDB ]
                            ↓
                    [ CloudWatch Logs/Alarms ]
```

### リソース仕様

| サービス | 仕様 | 理由 |
|---------|------|------|
| EC2 | t3.micro (2vCPU, 1GB) | 予算制約下で300名対応 |
| DynamoDB | オンデマンドモード | 大会時のみトラフィック増のため従量課金が有利 |
| CloudWatch | Logs + Alarms | 構造化ログ保存・異常検知 |
| Route53 | ホストゾーン1個 | 独自ドメイン運用 |
| VPC | デフォルトVPC利用 | コスト削減 |

### コスト見積もり（月額）

- EC2 t3.micro: ~$8 (750時間無料枠)
- DynamoDB: ~$5 (書き込み少量想定)
- CloudWatch: ~$3 (ログ保存5GB)
- Route53: ~$1
- **合計**: ~$17/月（無料枠適用後）

### スケーリング戦略

| 規模 | 対応策 |
|------|--------|
| 16-100名 | 単一EC2で十分 |
| 100-300名 | ALB + EC2オートスケーリング（最大3台） |
| 300名以上 | 将来検討（現在は対象外） |
